# GPLogListForm 测试说明

## 一、概述
GPLogListForm 是根据《SG1210v2.5表单设计.md》第五章"日志查询窗体"的设计要求实现的**全新**日志浏览窗体。

**重要说明**：
- LogListForm 是一个独立的新窗体，使用新的 Form ID：`WID_LogListForm`
- 替代了旧的 EventBrowserForm（`WID_EventListForm`）
- EventBrowserForm 已从项目中移除

## 二、实现的功能

### 2.1 基本功能
1. **日志分类显示**
   - 支持三种日志类型：事件(mltEvent)、告警(mltAlarm)、故障(mltFault)
   - 左侧垂直Caption显示当前日志分类名称
   - 底部显示当前项/总数索引（如："3:15"）

2. **日志列表显示**
   - 采用手动ownerdraw方式绘制列表
   - 每行显示日志时间戳和事件描述
   - 支持可见区域内的滚动显示
   - 选中项高亮显示（渐变背景）

3. **滚动条**
   - 当日志数量超过可见区域时显示滚动条
   - 滚动条位置反映当前浏览位置

### 2.2 交互操作

#### 2.2.1 键盘操作
1. **UP键**：向上移动选择
   - 在可见范围内移动时，只刷新相关项
   - 到达顶部时向上滚动
   - 在首项时循环到末项

2. **DOWN键**：向下移动选择
   - 在可见范围内移动时，只刷新相关项
   - 到达底部时向下滚动
   - 在末项时循环到首项

3. **LEFT键**：切换到上一个日志类型
   - mltFault → mltAlarm → mltEvent → MainForm

4. **RIGHT键**：切换到下一个日志类型
   - mltEvent → mltAlarm → mltFault → WLGListForm

5. **ESC键**：返回MenuForm

### 2.3 刷新策略
- 每秒自动刷新一次日志计数
- 如果日志数量变化，重新加载列表
- 符合设计文档中"每秒刷新一次"的要求

## 三、测试步骤

### 3.1 进入窗体
1. 启动 SG1210Sim.exe
2. 从启动画面进入主界面
3. 按 Enter 键进入菜单
4. 选择"故障日志"菜单项（菜单项2）
5. 确认进入日志查询窗体，默认显示故障日志(mltFault)

### 3.2 测试日志浏览
1. **空列表测试**
   - 如果没有日志，应显示"无事件记录"提示信息

2. **有日志时测试**
   - 确认日志按时间倒序排列（最新的在前）
   - 确认每条日志显示格式：`YYYY-MM-DD HH:MM:SS:mmm\n寄存器名称 事件描述`

3. **导航测试**
   - 按UP/DOWN键，确认选择移动正常
   - 确认选中项有渐变高亮效果
   - 确认滚动到边界时能正确换行/滚动

4. **日志分类切换测试**
   - 按RIGHT键：事件 → 告警 → 故障 → 波形日志窗体
   - 按LEFT键：故障 → 告警 → 事件 → 主窗体
   - 确认Caption标签颜色随日志类型变化

5. **滚动条测试**
   - 如果日志超过5条（可见区域），确认滚动条显示
   - 滚动时确认滚动条位置同步更新

6. **索引显示测试**
   - 确认右下角显示"当前项:总数"格式
   - 移动选择时确认索引同步更新

### 3.3 性能测试
1. 在有大量日志的情况下测试
2. 确认滚动流畅，无明显卡顿
3. 确认内存正常释放（退出窗体后）
