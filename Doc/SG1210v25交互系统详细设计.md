### 1.3 GWindow与emWin的关系

1. GWindow及其子类只在逻辑上占有emWin显示区域
2. 所有GWindow共用一份emWin的显示资源
    - 顶层的Form占有全部emWin显示区域
    - 顶层的Dialog占有部分显示区域，与顶层的Form共用，Dialog具有优先使用权，被遮挡的Form不再刷新显示
3. emWin的两个显示层（Layer)都归顶层GWindow使用。

### 1.4 GWindow规范

1. 占据屏幕确定位置
    - Form占全屏区域
    - Dialog占部分区域
    - 所有Form呈Z序，最后创建的在最顶层
2. 具有4个接口
    - onInit：初始化Form
    - onClose：关闭Form，释放资源
    - onShow：显示Form
    - onMessage：消息传递。顶层Form具有优先权

### 1.5 GFormabase规范

1. 非模态
    - 可以由FormCenter创建，亦可在Form中创建
    - 可以多个Form共存，按Z序迭放；
    - 创建显示完毕后即返回不管。
2. 占满全屏
    - 独占全屏空间，遮挡其后全部Form
    - 没有边框
    - 不支持缩放和移动
3. emWin的两层（Layer）分配
    - 背景层：在Form活动期间，显示不随时间而变化的信息
      - 背景图，如：Bkg320x240
      - Text Label：如：Caption
    - 前景层：放置需要定时更新，或随外部交互信号而变化的信息
      - 动态图标： 随装置状态变化而改变的图标
      - 动态Label：定时刷新的数值、状态文字等
      - 交互控件：响应交互操作而产生信息变化的信息，如：Listview、Menu等

### 1.6 GDialog规范

1. Dialog是模态Form
    - 由Form创建，同一时刻只有一个Dialog，
    - 永远在最顶层，下层Form不再响应消息
    - 由Exec方法执行初始化、显示、直到交互结束
2. Dialog只有两个结果结束
    - onCancel：撤消交互，关闭
    - onAccept：接受交互，关闭
3. 提供Exec方法，接管初始化、显示、交互、等到onCancel或onAccept后结束

## 二、交互系统

### 2.1 基本规范

1. 交互系统上电后根据复位源（Reset source）决定开启哪个Form
   - GetRSTSrc方法可获取装置的上电原因
   - 模拟器无上电原因，不影响交互
   - 检测到看门启动而上电\[GetRSTSrc(RRS_IWDGRST) == ture], 打开Main Form
   - 非看门狗启动，打开Splash form
2. 装置空闲时，永远停在Main Form
3. 装置交互超时，返回Main Form
    - 超过设定时间没有检测到交互操作，即认为交互超时
    - 超时时限可以设定，初始化由宏定义确定，如：1分钟
    - 交互超时后，Form Center可直接初始化emWin、FormCenetr，重建Main Form
      - 若处于Dialog交互结果被放弃
      - 若在Config Form，内容有被修改时，需要呼叫DevIntf_ServiceCancel，放弃编辑结果
    - 交互超时作业结束后，没有新的交互操作，不得再次执行交互超时操作

### 2.2 交互的分类

### 2.2.1 展示型表单

1. 展示内容固定，可静态，可动态刷新
2. 3种退出方式
   - 由交互操作切到其它表单
   - 计时结束后切到其它表单
   - 操作超时后被切换
3. 表单
   - Splash Form：交互切出，主动计时切出，超时切出
   - About  Form：交互切出，超时切出
   - Main Form：  交互切出
   - Device Test Form：

### 2.2.2 菜单

1. 展示内容固定，无动态刷新
2. 2种退出方式：
   - 交互切出
   - 超时切出
3. 表单：
   - 各级Menu Form

### 2.2.3 列表查看

1. 形式固定，展示内容动态生成
2. 来自GListViewForm，
3. 2种退出方式：
   - 交互切出
   - 超时切出
4. 表单：
   - Datalist Form
   - FatalLog Form
   - SystemLog Form

### 2.2.4 列表编辑

1. 列表编辑继承自GConfigForm，与列表查看都来自于GListViewForm，特性同上；
2. 列表编辑，在开始逻辑前，要求启动GLoginDialog，核对口令；
3. 退出时，若有修改，要求确认是否保存。
4. 表单：
   - Logic Config Form
   - Device Config Form
   - Serial Config Form
   - Ethernet Config Form
   - Factory Setup Form

## 三、消息系统

### 3.1 消息的生命周期

```
┌──────────┐    ┌─────────────┐    ┌──────────────┐    ┌──────────────┐
│ 事件源    │───→│ 消息转换     │───→│ GFormCentra   │───→│ 当前Form     │
│ (Key/    │    │ (KeyEvent/  │    │ SendMsg/     │    │ pMsg回调     │
│  Touch/  │    │  TouchEvent/│    │ PostMsg      │    │              │
│  Timer/  │    │  SendMsg)   │    │              │    │              │
│  ISR)    │    │             │    │              │    │              │
└──────────┘    └─────────────┘    └──────────────┘    └──────────────┘
```

### 3.2 消息分类

#### 3.2.1 系统消息（0~31）

| 消息 ID | 名称 | 触发源 | 处理要求 |
|---------|------|--------|---------|
| `GM_PAINT` (1) | 重绘请求 | WM | 重绘脏区域 |
| `GM_CREATE` (2) | 窗口创建 | WM | 分配资源 |
| `GM_CLOSE` (3) | 关闭窗口 | WM/Form | 释放资源 |
| `GM_TOUCH` (12) | 触摸事件 | TouchISR | 坐标解析+动作分发 |
| `GM_KEYDOWN` (13) | 按键按下 | KeyISR | 即时响应(导航/确认) |
| `GM_KEYPRESS` (14) | 按键长按 | KeyISR | 重复操作(滚动/增减) |
| `GM_KEYUP` (15) | 按键释放 | KeyISR | 完成操作 |
| `GM_TIMER_TICK` (16) | 定时器 | GFormCentra::Tick() | 刷新数据/动画/超时 |
| `GM_FORM_ACTIVATED` (18) | Form激活 | OpenForm | 恢复状态/注册资源 |
| `GM_FORM_DEACTIVATED` (19) | Form失活 | OpenForm/PopForm | 暂停刷新/保存状态 |
| `GM_IDLE_EXPIRE` (22) | 空闲超时查询 | GUITick | 阻止超时则填Data.v>0 |

#### 3.2.2 控件消息（40~125）

| 消息 ID | 名称 | 用途 |
|---------|------|------|
| `GM_LISTITEMS_CHANGED` (40) | 列表项变更 | ListView数据更新 |
| `GM_NEXT_PAGE` (50) | 下一页 | 翻页控件 |
| `GM_PERIOR_PAGE` (51) | 上一页 | 翻页控件 |
| `GM_INIT_DIALOG` (109) | 对话框就绪 | Dialog显示前回调 |
| `GM_SET_FOCUS` (110) | 设置焦点 | 焦点管理 |
| `GM_EDITOR_ACCEPT` (120) | 编辑器确认 | 数值/文本输入完成 |
| `GM_EDITOR_CANCEL` (121) | 编辑器取消 | 数值/文本输入放弃 |
| `GM_CLOSE_QUERY` (130) | 关闭询问 | 确认是否可关闭 |

#### 3.2.3 业务消息（200+）

| 消息 ID | 名称 | 用途 |
|---------|------|------|
| `GM_SETTING_ENABLE` (200) | 允许设置 | 解锁配置编辑 |
| `GM_SETTING_DISABLE` (201) | 禁止设置 | 锁定配置编辑 |
| `GM_DEVICE_COMFIRM` (202) | 设备确认 | 设备配置确认保存 |
| `GM_DEVICE_CANCEL` (203) | 设备取消 | 设备配置取消 |
| `GM_SETTING_COMFIRM` (204) | 配置确认 | 逻辑配置确认保存 |
| `GM_SETTING_CANCEL` (205) | 配置取消 | 逻辑配置取消 |
| `GM_TIMERECV` (220) | 时间接收 | RTC时间同步 |
| `GM_DATARECV` (221) | 数据接收 | 串口/网络数据到达 |

#### 3.2.4 自定义消息（≥ `GM_USER(10000)`）

各 Form 可定义私有消息，值从 `GM_USER` 开始。建议每个 Form 预留 100 个号段：

```cpp
// 示例：MainForm 自定义消息
#define GM_MAIN_REFRESH_VOLTAGE   (GM_USER + 0)
#define GM_MAIN_REFRESH_CURRENT   (GM_USER + 1)
#define GM_MAIN_REFRESH_POWER     (GM_USER + 2)
#define GM_MAIN_BATTERY_LOW       (GM_USER + 10)
#define GM_MAIN_INVERTER_FAULT    (GM_USER + 11)
```

### 3.3 消息路由规则

1. **定向投递**：`SendMsg` / `PostMsg` 仅投递给栈顶 Form（当前活跃 Form）
2. **广播**：`BroadcastMsg` 从栈底到栈顶依次投递给所有 Form
3. **Deferred 投递**：`PostMsg` 在下一个 `Tick()` 周期投递，ISR 安全
4. **对话框拦截**：Dialog 激活期间，下层 Form 不收任何消息（由栈顶机制自然保证）

### 3.4 消息处理优先级

每个 Form 的 `_OnMessage()` 应按以下顺序 dispatch：

```
1. GM_CLOSE_QUERY      — 询问能否关闭（最高优先）
2. GM_FORM_ACTIVATED   — 激活通知
3. GM_FORM_DEACTIVATED — 失活通知
4. GM_TIMER_TICK       — 定时刷新
5. GM_KEYDOWN/PRESS/UP — 按键
6. GM_TOUCH            — 触摸
7. GM_PAINT            — 重绘
8. 业务消息             — 自定义/配置消息
9. default: break      — 忽略未知消息
```

## 四、导航系统

### 4.1 导航栈模型

```
         Push(A)       Push(B)       Push(C)       Pop()
[]  ──────────→ [A] ──────────→ [A,B] ──────────→ [A,B,C] ──────→ [A,B]
                                  ↑                            ↓
                                  └──── Replace(D) ────→ [A,D] (Pop B + Push D)
```

### 4.2 三种导航操作

| 操作 | 行为 | 当前Form回调 | 新Form回调 | 使用场景 |
|------|------|-------------|-----------|---------|
| `PushForm` | 压栈+显示 | — | Init→Show | 进入子菜单、打开配置 |
| `ReplaceForm` | 弹当前+压新 | Close | Init→Show | 超时回首页、模式切换 |
| `PopForm` | 弹当前+恢复前一个 | Close | Show | 返回、ESC、取消 |

### 4.3 导航约束

1. **栈底保护**：栈深度≤1 时，`PopForm` 不执行（不能弹空）
2. **最大深度**：栈深度上限 16（`kMaxStack`），超过拒绝 Push
3. **重入保护**：同一 Form 不在栈上重复出现（通过 `IsFormOnStack` 检查，若已在栈上则 Pop 到该 Form）
4. **Main Form 锚定**：任何时刻栈底必须是 Main Form（除 Splash 阶段外）

### 4.4 典型导航路径

```
上电(Splash) ──(Enter/超时)──→ MainForm
                                  │
                    ┌─────────────┼─────────────┐
                    ↓             ↓             ↓
                MenuForm     DevTestForm    (状态区)
                    │
          ┌─────────┼─────────┬──────────┬──────────┐
          ↓         ↓         ↓          ↓          ↓
      EventList  Config   DevInfo   WavelogList  About
       (查看)   (编辑)    (查看)     (查看)      (查看)
                    │                    │
                    ↓                    ↓
              LoginDialog          WavelogView
                    │
                    ↓
              Editor(数值)
```

### 4.5 导航与消息的交互

```
PushForm(id):
  1. 锁定互斥量
  2. 如果栈非空，发 GM_FORM_DEACTIVATED 给当前 Form
  3. 新 Form 入栈
  4. 调用 pInit(para)
  5. 调用 pShow(para)
  6. 发 GM_FORM_ACTIVATED 给新 Form
  7. 解锁

PopForm():
  1. 锁定互斥量
  2. 如果栈深≤1，解锁并返回
  3. 发 GM_FORM_DEACTIVATED 给当前 Form
  4. 调用 pClose(para)  ← 释放资源
  5. 弹栈
  6. 调用下层 Form 的 pShow(para)
  7. 发 GM_FORM_ACTIVATED 给下层 Form
  8. 解锁
```

```

  │
  ├── 检测到按键/触摸 ──→ 更新 uKeyIdle = now + 120s
  │

  │     │

  │     │

  │
  └── gfc::Tick() → 投递 GM_TIMER_TICK
```

## 六、Dialog 模态机制设计

### 6.1 设计目标

当前系统所有 Form 以非模态方式管理。需要在不修改 GFormCentra 核心的前提下，为 Dialog 实现模态行为：
- Dialog 显示时，下层 Form 不响应交互
- Dialog 只有 Accept/Cancel 两种退出方式
- 提供 `Exec()` 风格的阻塞式调用（仅限 Sim）；MCU 用回调式

### 6.2 实现方案：半模态适配层

```cpp
// GDialogBase.h — 为需要模态行为的 Form 提供基类行为
namespace gdlg {

/// Dialog 结果
enum class Result : uint8_t {
    kNone,      // 尚未结束
    kAccepted,  // 用户确认
    kCancelled, // 用户取消
};

/// Dialog 状态（嵌入到每个 Dialog Form 的上下文中）
struct State {
    Result  result;
    bool    active;
    void*   userData;    // 传入传出参数
};

/// 标准 Dialog 消息处理模板
/// 在 _OnMessage 中调用，返回 true 表示消息已被拦截
bool HandleDialogMessage(GM_MESSAGE* pMsg, State* state,
                         TGWVoidProc onAccept, TGWVoidProc onCancel);

}  // namespace gdlg
```

### 6.3 Dialog 消息拦截规则

```
Dialog HandleMessage:
  │
  ├── GM_KEYUP + KEY_ENTER  → state.result = kAccepted; PopForm()
  ├── GM_KEYUP + KEY_ESC    → state.result = kCancelled; PopForm()
  ├── GM_TOUCH + Accept区域  → state.result = kAccepted; PopForm()
  ├── GM_TOUCH + Cancel区域  → state.result = kCancelled; PopForm()
  ├── GM_IDLE_EXPIRE        → 阻止超时 (Data.v = 1)
  │
  └── 其他消息: 正常处理（输入、重绘等）
```

### 6.4 Dialog 生命周期

```
创建:
  gfc::PushForm(WID_LoginDialog, &dialogParam)
    → _Init(): 初始化输入控件，设置 state.result = kNone
    → _Show(): 绘制 Dialog 框体（带半透明遮罩）

交互:
  每帧 _OnMessage():
    → 处理 GM_TOUCH / GM_KEYDOWN → 更新控件状态
    → 处理 GM_KEYUP + Enter/Esc → 设置 result，调用 PopForm()

退出:
  _Close():
    → 如果 result == kAccepted: 将输入数据写回 dialogParam.userData
```

## 七、显示层策略

### 7.1 emWin 双层分配

```
Layer 0 (背景层) — 非活动 Form 的背景保持
  ├── 背景位图 (如 Bkg320x240)
  ├── 静态 Label (标题、不可变文本)
  └── 刷新条件: 仅在 pShow 时绘制一次

Layer 1 (前景层) — 活动 Form 的交互内容
  ├── 动态图标 (状态指示灯、运行/停止动画)
  ├── 动态数值 (电压、电流、功率 — 每秒刷新)
  ├── 交互控件 (ListView、编辑器、菜单光标)
  └── 刷新条件: GM_TIMER_TICK 或交互消息触发
```

### 7.2 各 Form 的层使用规划

| Form | Layer 0 (静态) | Layer 1 (动态) |
|------|---------------|----------------|
| SplashForm | Logo 图片 | 进度动画 |
| MainForm | 背景图 + 分区标题 | 实时数据(电压/电流/功率)、状态图标 |
| MenuForm | 菜单框架/标题 | 高亮光标、选中标记 |
| EventListForm | 表头 | 事件列表滚动区 |
| ConfigForm | 配置项标签 | 编辑光标、当前值 |
| LoginDialog | (遮罩50%黑色) | Dialog框体、输入控件 |

### 7.3 层切换时机

```
Form激活(OnShow):
  1. 绘制Layer 0内容（一次性）
  2. 初始化Layer 1内容
  3. 启动定时刷新

Form失活(OnClose/Deactivate):
  1. 停止定时刷新
  2. Layer 1内容被新Form覆盖
  3. Layer 0内容保留在显存中（如需要）
```

### 9.2 列表查看 Form 模板（EventList / FatalLog / SystemLog / WavelogList）

```cpp
// GListViewForm 公共行为
static struct {
    uint16_t  cursor;       // 当前选中行 (0-based)
    uint16_t  topIndex;     // 可见区域第一行
    uint16_t  visibleRows;  // 可见行数 (通常 6-8)
    uint16_t  totalItems;   // 列表总项数
    bool      needRefresh;  // 数据变更标记
} s_listCtx;

// 按键导航
static void _OnKeyDown_List(uint16_t key) {
    switch (key) {
    case KEY_UP:
        if (s_listCtx.cursor > 0) {
            s_listCtx.cursor--;
            if (s_listCtx.cursor < s_listCtx.topIndex) {
                s_listCtx.topIndex = s_listCtx.cursor;
            }
        }
        break;
    case KEY_DOWN:
        if (s_listCtx.cursor < s_listCtx.totalItems - 1) {
            s_listCtx.cursor++;
            if (s_listCtx.cursor >= s_listCtx.topIndex + s_listCtx.visibleRows) {
                s_listCtx.topIndex = s_listCtx.cursor - s_listCtx.visibleRows + 1;
            }
        }
        break;
    case KEY_ENTER:
        // 查看选中项的详情
        gfc::PushForm(WID_WLGViewForm, &s_listCtx.cursor);
        break;
    }
}

// 列表绘制（虚拟滚动）
static void _DrawList(void) {
    GUI_SetLayer(1);
    for (int row = 0; row < s_listCtx.visibleRows; row++) {
        size_t itemIdx = s_listCtx.topIndex + row;
        if (itemIdx >= s_listCtx.totalItems) break;

        bool isSelected = (itemIdx == s_listCtx.cursor);
        uint32_t bgColor = isSelected ? GUI_BLUE : GUI_BLACK;
        uint32_t fgColor = isSelected ? GUI_WHITE : GUI_LIGHTGRAY;

        GUI_SetBkColor(bgColor);
        GUI_SetColor(fgColor);
        // 绘制该行文本
        GUI_DispStringAt(GetItemText(itemIdx), 10, 30 + row * 28);
    }
}
```

```cpp
        }
        break;
```

```cpp
        pMsg->Data.v = 1;  // Dialog 不响应空闲超时
        break;
    }
}
```

## 十、错误处理与恢复

### 10.1 看门狗复位处理

```
上电 → GUIStart()
  │
  ├── GetRSTSrc(RRS_IWDGRST) == true:
  │     → 跳过 Splash，直接进 MainForm
  │     → FGUIState.uStartupTime |= 1
  │     → 显示 "Watchdog Reset" 提示（3秒后消失）
  │
  └── GetRSTSrc(RRS_IWDGRST) == false:
        → 正常启动，显示 SplashForm
```

### 10.2 致命错误显示

```
GUIShowFatalMessage(const char* msg):
  │
  ├── 直接调用 FFatalForm.pShow(msg)  ← 绕过 GFormCentra 导航栈
  │   （FatalForm 占有全部显示资源，不响应导航）
  │
  └── FatalForm 显示:
        ├── 红色背景
        ├── 错误图标（感叹号）
        ├── 错误文本
        └── "长按 Enter 重启" 提示
```

### 10.3 导航异常恢复

```
情况1: 栈意外为空
  → GUIFormClose 检测到 IsStackEmpty()
  → 自动 PushForm(WID_MainForm)

情况2: 注册表查找失败
  → FindForm 返回 nullptr
  → 调用方检查，降级到 MainForm

情况3: 消息队列满
  → PostMsg 静默丢弃（队列满时 return）
  → 适用于非关键消息（关键消息走 SendMsg）
```

### 12.2 配置编辑流程

```
MainForm → 按键选择"配置" → PushForm(MenuForm)
  │
MenuForm → 选择具体配置项 → PushForm(ConfigForm)
  │
ConfigForm._Show:
  ├── [未授权] → PushForm(LoginDialog)
  │                 ├── 输入密码 → Enter
  │                 │     ├── 正确: PopForm → ConfigForm 重新 _Show
  │                 │     └── 错误: 显示错误提示
  │                 └── ESC → PopForm → 回到 MenuForm
  │
  └── [已授权] → 显示配置列表
        │
        ├── ↑↓ 选择配置项
        ├── Enter → 进入编辑模式
        │     ├── ↑↓ 改变值
        │     ├── ←→ 切换编辑位
        │     ├── Enter → GM_EDITOR_ACCEPT → 保存值 → 标记isModified
        │     └── ESC   → GM_EDITOR_CANCEL → 恢复原值
        │
        └── ESC 退出:
              ├── isModified == true:
              │     └── 弹出确认 Dialog
              │           ├── "保存" → DevIntf::WriteConfig() → PopForm
              │           ├── "放弃" → DevIntf::ServiceCancel() → PopForm
              │           └── "取消" → 留在 ConfigForm
              └── isModified == false:
                    └── 直接 PopForm
```

### 12.3 空闲超时返回流程

```
用户操作 → 更新 uKeyIdle
  │
  │ (无操作持续120秒)
  │
GUITick:
  ├── 检测到超时
  ├── 关闭 LCD 背光
  │
  ├── 当前 Form = MainForm?
  │     └── Yes: 只关背光，不做其他操作
  │
  └── 当前 Form ≠ MainForm?
        ├── 发 GM_IDLE_EXPIRE 给当前 Form
        │     ├── Form 是 Dialog? → Data.v = 1 (阻止)
        │     └── Form 是编辑且 isModified? → Data.v = 1 (阻止)
        │
        └── Form 未阻止:
              └── ReplaceForm(WID_MainForm)
                    ├── 当前 Form._Close (放弃所有编辑)
                    └── MainForm._Show (恢复显示)
```
